## 项目概述

**知喻**是一个面向高校課后学习场景的**多智能体协同平台**，核心理念为 **"知识AI化，价值真人化"**。

- **AI 承接**：标准化知识传递、基础答疑、习题批改、知识点抽背
- **真人回归**：经验分享、价值引导、学情把控、个性化辅导

通过一个**教师 + AI教师 + 分层AI学生群体 + 真人学生**的四层互动架构，模拟真实课堂讨论氛围，降低学生提问门槛，辅助教师精准掌握学情。

## 1. 双角色账号权限体系

### 1.1 用户角色定义

| 角色 | 定位 | 注册方式 |
|------|------|----------|
| **老师（teacher）** | 课程创建者与管理者 | 自行注册，选择"老师"身份 |
| **学生（student）** | 课程学习者 | 自行注册，选择"学生"身份 |

> 面向高校课后场景设计，无需对接学校统一身份认证，本地轻量化注册登录。

### 1.2 老师权限全景

| 权限项 | 说明 |
|--------|------|
| 上传课件 | 支持 PDF/Word/PPT，自动触发 MinerU 解析 |
| 管理课程知识库 | 查看已索引文档、删除文档、重建索引 |
| 生成 AI 老师 | 基于课件内容 + 授课风格，自动生成 AI 教师角色卡 |
| 管理 AI 学生 | 添加/移除/配置 AI 学生角色（基础/中等/拓展/学长） |
| 创建课程群 | 设置课程名称、描述、知识库绑定 |
| 查看学情数据 | 高频问题 TOP10、薄弱知识点分布、班级活跃度 |
| 导出学情报告 | 按周生成 PDF 报告 |
| 审批入班申请 | 查看学生申请列表，批准或拒绝 |
| 出题 | 基于知识库生成练习题（选择/简答混合） |

### 1.3 学生权限全景

| 权限项 | 说明 |
|--------|------|
| 申请加入课程 | 发送入班申请，等待老师审批 |
| 参与群聊对话 | 与 AI 教师、AI 学生、其他真人学生互动 |
| 练习题库 | 参加老师生成的练习题，查看批改结果 |
| 查看错题本 | 自动记录所有答错题目，按课程分类查看 |
| 知识点抽背 | 随机抽取知识点进行问答 |
| 查看思维导图 | 查看课程知识脉络 |
| 查看个人学习记录 | 练习历史、正确率趋势 |
| **不能** | 修改任何 AI 设定、知识库内容、课程配置 |

### 1.4 权限校验机制

```
请求到达 → 提取 JWT Cookie → 解码获取 user_id + role
         → 查询数据库验证用户有效性
         → 根据路由权限标签（require_teacher / require_user）判断
         → 合法：放行 / 不合法：返回 403 或重定向到登录页
```

- 所有角色编辑、知识库修改、数据导出接口强制校验 `require_teacher`
- 学生账号访问管理接口直接返回 `403 Forbidden: 仅老师可执行此操作`
- JWT Token 有效期 7 天，支持自动续期

## 2. 增强型课程 RAG 知识库系统

### 2.1 文档预处理管道

```
上传课件（PDF/Word/PPT）
        │
        ▼
   ┌──────────── MinnerU Docker 解析 ──────────┐
   │  - 提取标题层级（# ## ###）               │
   │  - 提取公式 → 转为 LaTeX 格式             │
   │  - 提取表格 → 转为 Markdown 表格          │
   │  - 提取正文 → 分段结构化 Markdown         │
   │  - 自动过滤页眉/页脚/页码                 │
   └──────────────────┬──────────────────────┘
                      │
                      ▼
           结构化 Markdown 文本
                      │
                      ▼
           文本分块（按章节边界 + 语义分块）
           单块 500-1000 字符，块间重叠 100 字符
                      │
                      ▼
           百炼 text-embedding-v3 → 向量化
                      │
                      ▼
           ChromaDB 存储（按 course_id 隔离）
```

**回退机制**：当 MinerU 服务不可用时，自动降级为 PyMuPDF（纯 Python PDF 解析）+ python-docx（Word 解析）+ python-pptx（PPT 解析），保证基本功能不受影响。

### 2.2 二阶检索架构

```
学生提问："什么是贝叶斯定理？"
        │
        ▼
   ┌── 第一步：向量召回 ──────────────────┐
   │  查询 → 百炼Embed → ChromaDB 相似度  │
   │  搜索 → 召回 Top 20 相关片段         │
   └──────────────────┬──────────────────┘
                      │
                      ▼
   ┌── 第二步：精排 ──────────────────────┐
   │  20个候选片段 → 百炼 Rerank API      │
   │  → 按相关性分数重排 → 取 Top 5      │
   └──────────────────┬──────────────────┘
                      │
                      ▼
           Top 5 片段 → 注入 AI 教师回答 prompt
                      │
                      ▼
           百炼 Qwen → 生成最终回答
```

### 2.3 知识库绑定与约束

- 每个课程的知识库与对应的 AI 老师角色**深度绑定**
- 仅在该课程群聊内可调���知识库
- 学生端**无法导出、下载知识库原始内容**
- AI 教师回答必须**优先基于知识库内容**
- 超出知识库范围需明确标注 **「课外拓展内容，仅供参考」**
- **禁止**编造课程评分标准、考试重点、教师偏好等未在知识库中明确记载的信息

## 3. 分层 AI 角色自动生成体系

### 3.1 AI 教师角色生成

**触发条件**：老师上传课件后，在教学风格设置页面填写偏好，点击"生成 AI 教师"。

**生成流程**：
```
课件内容 + 教学风格描述
        │
        ▼
  ┌── 百炼 Qwen ──────────────────────────┐
  │  生成标准化角色卡 JSON：               │
  │  - name: AI教师名称                    │
  │  - personality: 人设定位（如"精通微积  │
  │    分的耐心教师"）                     │
  │  - teaching_style: 教学风格            │
  │  - dialogue_style: 对话风格           │
  │  - greeting: 开场白                    │
  │  - knowledge_boundary: 知识边界        │
  │  - world_book: 教学原则条目            │
  └──────────────────┬──────────────────┘
                     │
                     ▼
  自动绑定课程知识库 → 加入课程群
```

### 3.2 分层 AI 学生角色体系

| 类型 | 角色名 | 定位 | 提问风格 | 功能目标 |
|------|--------|------|----------|----------|
| **基础薄弱型** | 基础小问 😊 | 进度较慢，从最基本概念开始理解 | 语气谦逊，常用"这个我不太懂""能再解释一下吗" | 降低提问门槛，带动基础讨论 |
| **中等巩固型** | 中坚小固 📚 | 基础较好，关注考点和易错点 | 认真务实，喜欢追问"这个会考吗" | 贴合应试需求，强化记忆 |
| **拓展思考型** | 拓思考 💡 | 基础扎实，追求深层原理 | 带有思辨性，常用"这是为什么呢" | 引导深度思考，延伸讨论 |
| **AI 学长/学姐** | 学长知喻 🎓 | 经验丰富，熟悉考试规律 | 亲切老练，乐于分享"往年经验" | 学习方法指导 + 应试技巧 |

### 3.3 发言规则配置

老师可配置每位 AI 学生的：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 活跃度 | 每 N 条消息后自动发言的概率 | 0.3（30%） |
| 触发规则 | 轮序发言 / @触发 / 话题关键词触发 | 混合模式 |
| 话题范围 | 允许该角色讨论的知识域 | 绑定课程 |
| 发言间隔 | 两次自动发言之间的最小消息数 | 3 条 |

## 4. 应试学习工具集

### 4.1 流程总览

```
老师端                   学生端
  │                       │
  ├── 生成练习题 ────────→ 参加答题
  │    基于知识库出题      │
  │    选择/简答混合       ├── 提交批改
  │    可指定难度/章节     │    选择→自动对比
  │                       │    简答→AI评阅
  │                       │
  │                       ├── 查看结果
  │                       │    得分+逐题反馈
  │                       │
  │                       ├── 错题自动记录
  │                       │    关联知识点
  │                       │
  │                       ├── 知识点抽背
  │                       │    随机出题+AI判分
  │                       │
  │                       └── 思维导图
  │                            自动生成Markdown
  ▼                       ▼
```

### 4.2 题库生成

- 调用百炼 Qwen，从知识库中提取关键知识点
- 支持混合出题：选择题（4 选项）+ 简答题
- 支持 3 级难度：easy / medium / hard
- 每道题自动标注所属知识点
- 老师可指定出题数量（5/10/15/20）和难度倾向

### 4.3 自动判分

| 题型 | 判分方式 | 响应时间 |
|------|----------|----------|
| 选择题 | 精确匹配正确答案，大小写不敏感 | 即时 |
| 简答题 | 调用百炼 Qwen，根据参考答案判断核心观点正确性 + 生成评语 | 1-3 秒 |

### 4.4 错题本

- 所有答错的题目**自动记录**
- 按课程分类查看
- 每条记录包含：题目原文、正确答案、你的答案、知识点标签
- 支持单条删除
- 可作为抽背的来源

### 4.5 知识点抽背

- 从知识库中随机抽取知识点，以问答形式展示
- 学生输入答案后，AI 即时评判并给出参考答案
- 记录已抽取的知识点，避免重复
- 可看答案后再进入下一题

### 4.6 思维导图

- 从课程知识库提取章节层级结构
- 调用百炼 Qwen 分析知识点之间的**父子关系与层级结构**
- 生成 Markdown 格式的层级大纲（`# ## ### -`）
- 可直接复制到 Markmap / XMind 等工具可视化

## 5. 学情驱动的动态角色迭代

### 5.1 对话数据采集

```
课程群聊中的每一条消息
        │
        ▼
  ┌── 自动归档 ──────────────────────────┐
  │  存入数据库（对话表）                 │
  │  每条消息标注：角色（AI/真人）、       │
  │  知识点标签（从知识库匹配）           │
  └──────────────────┬──────────────────┘
                     │
                     ▼
  ┌── 分类统计 ──────────────────────────┐
  │  - 高频疑问统计                      │
  │  - 常见误区识别                      │
  │  - 知识点覆盖度                      │
  │  - 学生参与度                        │
  └──────────────────────────────────────┘
```

### 5.2 动态角色迭代（7 天周期）

每 7 天自动执行一次：

```
步骤1：汇总过去 7 天对话数据
        │
        ▼
步骤2：调用百炼 Qwen 分析
       ├── 班级共性问题是哪些？
       ├── 哪些知识点学生普遍困惑？
       ├── AI 学生的提问是否贴合实际学情？
        │
        ▼
步骤3：自动更新 AI 学生角色 prompt
       ├── 基础小问 → 加入本周高频迷惑点
       ├── 中坚小固 → 加入易错题知识点
       └── 拓思考 → 加入延伸讨论方向
        │
        ▼
步骤4：生成本周学情报告摘要
```

### 5.3 学情报告（PDF 导出）

每周自动生成，包含：

| 板块 | 内容 |
|------|------|
| **高频问题 TOP10** | 按出现次数排序，附知识点标签 |
| **知识薄弱点分布** | 以知识点为维度的正确率统计 |
| **班级活跃数据** | 消息总数、参与学生数、AI 发言占比 |
| **趋势对比** | 与上周对比的改善/恶化趋势 |

```
Course（课程）
├── id, name, description

└── knowledge_base_id, created_at

├── id, course_id → Course

├── id, course_id → Course
├── filename, file_path, file_type
└── chunk_count, uploaded_at

Quiz（练习）
├── id, course_id → Course
└── title, question_count, created_at

Question（题目）
├── id, quiz_id → Quiz
├── content, type: choice / short_answer
├── options(JSON), correct_answer
└── knowledge_point, difficulty

QuizAttempt（答题记录）
├── id, quiz_id → Quiz, student_id → User
└── score, total, completed_at

Answer（单题回答）
├── id, attempt_id → QuizAttempt
├── question_id → Question
└── student_answer, is_correct, feedback

WrongBookRecord（错题记录）
├── id, student_id → User, course_id → Course
├── question_content, correct_answer, student_answer
└── knowledge_point, question_type, source_quiz_id
```

## 竞赛价值定位

### 创新点

| 创新点 | 说明 |
|--------|------|
| **分层 AI 学生群体** | 不是简单的"AI 助教"，而是模拟真实课堂中的不同类型学生，带动讨论氛围 |
| **学情驱动角色迭代** | AI 角色不是静态的，而是根据真实的班级学情数据定期更新，越用越精准 |
| **教育场景 RAG 二阶检索** | 召回 20 → 精排 5，远超简单关键词搜索的质量 |
| **老师留白，AI 填充** | 真人老师从重复答疑中解放，专注于 20% 的高价值教学工作 |

### 与现有产品的差异
